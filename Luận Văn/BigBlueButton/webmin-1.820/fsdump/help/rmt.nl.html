Pad naar rmt op het remote systeem
Om met een tar opdracht een backup te schrijven naar een remote server moet deze de rmt opdracht aanroepen op het bestemming systeem. Dit kan normaal gesproken op Standaard blijven staan, maar wanneer de bron en de bestemming server verschillende besturingssystemen gebruiken kan het nodig zijn dat u het juiste pad naar rmt moet specificeren op de bestemming server. Alsmede gemeenschappelijke paden bijvoegen zoals /etc/rmt, /usr/libexec/rmt en /usr/sbin/rmt.
